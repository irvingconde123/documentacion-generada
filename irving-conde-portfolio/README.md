# Irving Conde Portfolio - Base de conocimiento

Esta carpeta documenta problemas, decisiones y soluciones vistas durante la estabilizacion del portafolio `irving-conde-portfolio`.

## Estructura

- `errores/`: sintomas, causas probables, evidencia y prevencion.
- `soluciones/`: fixes aplicados, arquitectura observada y criterios de mantenimiento.
- `ideas/`: proyectos propuestos para ampliar el portafolio con evidencia tecnica.

## Contexto del proyecto

- Aplicacion Angular/Ionic estatica.
- Demos sanitizadas para Adastra, Landing, CMS y Hostlyc.
- Casos de estudio con narrativa de arquitectura, decisiones y evidencia.
- Despliegue en GitHub Pages y Vercel.

## Regla de mantenimiento

Cada vez que se corrija un bug visual o de interaccion, registrar:

1. Sintoma observable.
2. Causa tecnica.
3. Archivos tocados.
4. Prueba manual o automatizada.
5. Como prevenir la regresion.
