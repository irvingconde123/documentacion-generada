# Plantillas de trabajo reutilizables

## 1. Plantilla de PR

### Título

[Tipo]: [Resumen breve]

### Descripción

- Qué se cambió
- Por qué se hizo
- Impacto esperado
- Riesgos o bloqueadores

### Checklist

- [ ] Se han revisado los cambios
- [ ] Se han actualizado los documentos relevantes
- [ ] Se han validado pruebas o verificaciones básicas
- [ ] No hay cambios sensibles sin aprobación

## 2. Plantilla de commit

### Formato recomendado

`tipo(scope): descripción`

Ejemplos:
- `feat(api): agregar endpoint de inventario`
- `fix(frontend): corregir scroll interno`
- `docs(governance): actualizar reglas de trabajo`

## 3. Plantilla de issue o tarea

### Título

[Resumen breve]

### Contexto

- Qué problema o necesidad existe
- Qué se espera lograr

### Criterios de aceptación

- [ ] Se define el alcance claro
- [ ] Se documenta el impacto esperado
- [ ] Se identifica la dependencia si aplica

## 4. Plantilla de estado de proyecto

### Resumen general

- Estado actual
- Riesgos principales
- Próximos pasos

### Áreas críticas

- Arquitectura
- Integraciones
- Calidad
- Seguridad
- Operación
