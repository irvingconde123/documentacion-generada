# Gobernanza de proyectos

## Propósito

Esta carpeta centraliza las reglas, plantillas y prácticas reutilizables para la gestión de proyectos de software. Su objetivo es servir como fuente de conocimiento transversal para cualquier proyecto nuevo, sin depender de la arquitectura específica de uno solo.

## Qué incluye

- reglas de trabajo por agentes
- reglas de calidad y límites de archivo
- plantillas de PR y commits
- gestión de contratos compartidos
- estrategia de ramas y ambientes
- seguimiento de estado del proyecto
- reglas para documentación y coordinación

## Cómo usarla

- Copiar o adaptar estos documentos cuando se inicie un nuevo proyecto.
- Mantenerlos como base de gobierno operativo.
- Actualizarlos cuando se detecten mejoras compartidas entre proyectos.
